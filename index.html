Theory

The Algorithm

LL(1) parsing is the backbone of predictive, top-down parsing with one token of lookahead.

FIRST Sets

FIRST(α) is the set of terminals that can begin a string derived from α. If α ⟹* ε then ε ∈ FIRST(α).

FIRST(ε) = { ε }
FIRST(aβ) = { a } // terminal
FIRST(Aβ) = FIRST(A) – {ε}
if ε ∈ FIRST(A), add FIRST(β)

FOLLOW Sets

FOLLOW(A) is the set of terminals that can appear immediately to the right of A in any sentential form.

$ ∈ FOLLOW(S) // start symbol
A → αBβ : FIRST(β)–{ε} ⊆ FOLLOW(B)
A → αB or ε ∈ FIRST(β) :
FOLLOW(A) ⊆ FOLLOW(B)

Building the Parse Table

For each production A → α, for each terminal a ∈ FIRST(α), add A → α to M[A, a]. If ε ∈ FIRST(α), add A → α to M[A, b] for each b ∈ FOLLOW(A).

For A → α :
∀ a ∈ FIRST(α) – {ε} :
M[A, a] = A → α
if ε ∈ FIRST(α) :
∀ b ∈ FOLLOW(A) :
M[A, b] = A → α

LL(1) Condition

A grammar is LL(1) if and only if for every non-terminal A, every pair of distinct productions A → α and A → β satisfy:

1. FIRST(α) ∩ FIRST(β) = ∅
2. At most one of α, β derives ε
3. If β ⟹* ε :
FIRST(α) ∩ FOLLOW(A) = ∅

No left-recursive or ambiguous grammar is LL(1).
